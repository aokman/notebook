Houdini 的工程管理主要通过工程目录、环境变量和相对路径规范来实现的。

参考链接：[SideFX 官方文档：Project management](https://www.sidefx.com/docs/houdini/basics/project.html)

## 核心概念

Houdini 可以把 `.hip` 场景文件保存在任意位置，并通过环境变量来管理资源路径。最常用的是 `$HIP` 和 `$JOB`。

`$HIP` 表示当前 `.hip` 场景文件所在的目录。它适合引用和当前场景文件绑定较强的资源，例如：

```text
$HIP/camera_maps
```

`$JOB` 表示当前工程目录。对于包含多个场景文件的工程，Houdini 推荐使用 `$JOB` 作为统一的工程根目录，例如：

```text
$JOB/textures
$JOB/geo
$JOB/render
```

## 创建和设置工程

通过 `File > New Project` 可以创建一个新的工程目录。Houdini 会自动生成一组常用子目录，用来存放渲染输出、数字资产、脚本、模拟缓存等工程文件，同时把 `$JOB` 设置为该工程路径。

通过 `File > Set Project` 可以选择一个已有目录作为当前工程目录，也就是重新设置 `$JOB`。也可以通过 `File > Set Recent Project` 从最近使用的工程中选择。

需要注意的是，设置工程不会改变当前打开的 `.hip` 文件，只会改变 `$JOB` 的值。

## 路径管理方式

Houdini 鼓励使用相对工程路径，而不是硬编码绝对路径。这样可以让工程更容易移动、打包、协作，或者发送到远程机器进行模拟和渲染。

常见约定如下：

- `$JOB`：用于工程级共享资源，例如贴图、缓存、几何文件、HDA、脚本等。
- `$HIP`：用于当前场景文件附近的资源，适合场景专属文件。

在文件选择器中，可以点击左侧的 `$JOB` 快速跳转到工程目录。这样选择出来的路径会更容易保持为基于 `$JOB` 的相对路径。

## 检查外部引用

`Render > Preflight` 可以检查当前场景中是否存在工程外部引用，例如指向磁盘其他位置的绝对路径。

如果发现外部引用，Preflight 窗口会用红色标记。可以直接在窗口中选择这些引用，并编辑对应表达式，把它们改成基于 `$JOB` 或 `$HIP` 的路径。
